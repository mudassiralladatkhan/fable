# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| latest  | ✅        |
| < 1.0   | ❌        |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

We use GitHub's private vulnerability reporting. To report a vulnerability:

1. Go to the [Security tab](https://github.com/chasedputnam/go-kiro-gateway/security) of this repository
2. Click **"Report a vulnerability"**
3. Fill in the details and submit

You can expect:
- **Acknowledgement** within 48 hours
- **Status update** within 7 days
- **Resolution or mitigation** as quickly as possible depending on severity

## What to Include

Please include as much of the following as possible:

- Type of vulnerability (e.g. authentication bypass, injection, credential exposure)
- Full paths of affected source files
- Steps to reproduce
- Proof-of-concept or exploit code (if available)
- Impact assessment

## Scope

This policy covers the `go-kiro-gateway` application itself. It does not cover:

- Third-party dependencies (report those upstream)
- The Kiro/AWS API services this gateway proxies

## Preferred Languages

We prefer reports in English.
